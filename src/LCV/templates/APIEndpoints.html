Compatibility SPDX Verifier API

Enter inbound licenses using their SPDX IDs, separated by commas

Example of usage of the API methods /LicensesInput, /LicensesInputSPDX, and /GetGitHubOutboundLicense

  • http://127.0.0.1:3251/LicensesInput?InboundLicenses=Apache 2.0,MIT License&OutboundLicense=MIT
  • http://127.0.0.1:3251/LicensesInputSPDX?InboundLicenses=Apache-2.0,MIT&OutboundLicense=MIT

These two endpoints accept both POST and GET requests, and they can be used via browser or collaboration platform for API development, such as Postman.

The syntax is as follows:

  • /LicensesInput?InboundLicenses=FirstInboundLicense,SecondInboundLicense, ... nInboundLicense&OutboundLicense=UniqueOutboundLicenseSPDXid

  • /LicensesInputSPDX?InboundLicenses=FirstInboundLicenseSPDXid,SecondInboundLicenseSPDXid, ... nInboundLicenseSPDXid&OutboundLicense=UniqueOutboundLicenseSPDXid

This endpoint accepts a GitHub API URL and retrieves the SPDX ID of its outbound license.

  • /GetGitHubOutboundLicense?url=https://api.github.com/repos/hope-for/hope-boot/license
